01

前期准备

在开始之前，你需要准备以下内容：

✓
GitHub 账号

访问 github.com 注册一个免费账号

✓
代码编辑器

推荐使用 VS Code、Sublime Text 或其他你喜欢的编辑器

✓
基础 HTML/CSS 知识

了解基本的网页结构和样式设置

💡

提示：GitHub Pages 是免费的静态网站托管服务，非常适合个人博客、项目文档和作品集展示。

02

创建仓库

登录 GitHub 后，创建一个新的代码仓库：

1

点击右上角 [+] 按钮，选择 [New repository]

2

输入仓库名称，选择 Public（公开）权限

3

勾选 [Add a README file] 选项

4

点击 Create repository 完成创建

仓库命名规则 
个人主页模式: 用户名.github.io
项目子站模式: 任意名称（如 my-project）
03

配置 GitHub Pages

在仓库中启用 GitHub Pages 功能：

1

进入仓库页面，点击 [Settings]（设置）

2

在左侧菜单找到 [Pages] 选项

3

在 Source 部分选择 Deploy from a branch

4

选择 main 分支，目录选择 / (root)

5

点击 Save 保存设置

ℹ️

说明：配置完成后，GitHub 会自动为你生成一个访问链接，格式为：https://用户名.github.io/仓库名/

04

编写网站代码

创建你的第一个静态网页文件：

📁 项目结构

your-repo/
├── index.html      ← 首页（必须）
├── style.css       ← 样式文件
├── script.js       ← 脚本文件
└── docs/           ← 文档目录（可选）
    ├── page1.html
    └── page2.html
index.html 示例 
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>我的静态网站</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <h1>欢迎来到我的网站</h1>
    <p>这是我的第一个 GitHub Pages 网站</p>
</body>
</html>
05

部署上线

将代码推送到 GitHub，网站将自动部署：

Git 命令 
# 初始化本地仓库
git init

# 添加所有文件
git add .

# 提交更改
git commit -m "初始化网站"

# 关联远程仓库
git remote add origin https://github.com/用户名/仓库名.git

# 推送到 GitHub
git push -u origin main
🎉

完成！等待几分钟后，访问你的网站：
https://用户名.github.io/仓库名/

✓
访问测试

打开浏览器，输入你的网站地址，检查是否正常显示

✓
移动端适配

使用手机访问，确保网站在移动设备上也能正常显示
